📘 Introduction

Welcome to the SentiScan API and platform documentation. Here you'll learn how to integrate our sentiment and emotion analysis tools into your app or workflow.

🔐 Authentication

Each user has to create an api_key to use SentiScan. The api_key can be created under the APIs tab. This key must be sent in requests to access the API securely.

Example usage:

{ "api_key": "your-api-key-here", "text": "I'm happy today!" }

🧠 API Usage

Send a POST request to /analyze with your API key and input text or file.

You can also attach an optional file or URL (R3 only) for analysis. The response includes:

  • 🧾 Original content
  • 💡 Sentiment score & label
  • 🎯 Keywords extracted
  • 🎭 Emotion classification
  • 📈 Confidence score

🤖 Model Capabilities

R1

  • • Supports only .txt
  • • Basic sentiment & emotion
  • • Fastest processing
  • • Minimal output fields

R2

  • • Supports .txt and .pdf
  • • Improved keyword extraction
  • • Better emotion mapping
  • • Better than R1

R3

  • • Supports .txt, .pdf, .docx
  • • Allows URL scraping
  • • Advanced contextual detection
  • • Returns emotion intensity (1–10)

📊 Usage Limits

Each user has a free tier limit of 20 analyses per day.

⚠️ Usage warnings are shown at:

  • 🔸 50% usage → “50% usage reached”
  • 🔶 75% usage → “75% usage reached”
  • ❌ 100% usage → analysis blocked until next day

📂 Project-Level Analysis

You can analyze an entire project to get the overall emotional tone and polarity by aggregating all the content submitted using that project’s API keys.

Response includes:

  • • 🧠 Overall dominant emotion
  • • ⚖️ Overall polarity (Positive/Neutral/Negative)
  • • 📝 Summary description of the tone
